EcoPaste 剪切板信息采集版使用说明

一、用途
本版本在 EcoPaste 剪切板管理基础上，增加了结构化信息采集功能。
复制疑似企业、门店或客户资料后，程序会按设置自动提取以下字段：

公司名称
姓名/法人
电话号码
邮箱
地址

二、输出位置
默认保存目录：

规则提取：D:\信息采集\规则提取
AI 识别：D:\信息采集\AI识别

每个目录下会生成：

records.csv：采集结果表格
.state.json：去重状态文件

三、规则提取
规则提取完全在本地运行，不需要联网，也不需要 AI Key。
开启路径：

偏好设置 -> 剪切板 -> 结构化采集 -> 规则提取

规则提取会优先识别公司名称、法人/联系人、电话、邮箱、地址。
当前规则要求能识别出公司名称，并且至少有两个有效字段，才会写入表格。

四、AI 识别
AI 识别适合字段不规整、页面文字混杂、规则提取不够稳定的场景。
开启路径：

偏好设置 -> 剪切板 -> 结构化采集 -> AI 识别

需要填写：

接口地址：可填写第三方 OpenAI 兼容平台的 API 基地址、/v1 地址，或完整 chat/completions 地址
模型名：可手动填写，也可以点击“获取模型”拉取列表
API Key：第三方平台提供的密钥
超时时间：请求超过该时间会自动放弃，继续处理下一条

五、第三方 AI 平台注意事项
AI 接口需要尽量兼容 OpenAI Chat Completions 格式。
程序会发送 model、messages、temperature 字段。
鉴权方式为 Authorization: Bearer API_KEY。
返回内容需要包含 choices[0].message.content，并且内容最好是 JSON。

六、常见问题

1. 获取模型失败
请检查接口地址、API Key、网络、第三方平台是否支持 /models 接口。

2. 测试 AI 接口返回 404
通常是接口地址不对。建议先填写平台给出的 API 基地址或 /v1 地址。

3. 规则提取没有生成 records.csv
请确认“规则提取”开关已开启，并且复制的内容包含公司/门店名称和至少一个有效联系方式或地址。

4. AI 识别没有写入
请先点击“测试 AI 接口”，确认接口可用；再确认 AI 返回的是有效 JSON。

七、建议用法
如果复制来源比较固定，优先开启规则提取。
如果复制来源很杂，开启 AI 识别。
两个通道可以同时开启，结果会分别写入不同目录，不会混在一起。
